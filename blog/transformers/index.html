A Unified Mathematical Framework for Backpropagation, Attention, and State Management in Large-Scale Transformers

June 25, 2026 (1d ago)

Introduction

Modern large-scale neural networks, particularly Transformer-based architectures, are constrained not only by computational cost but also by the management of intermediate state during both training and inference.

This paper develops a unified mathematical view of these systems as instances of a single underlying problem: state management in computational graphs. In this view, training and inference are governed by how intermediate quantities are stored, recomputed, or distributed under memory and communication constraints.

We begin from first principles of reverse-mode automatic differentiation and derive the structure of backpropagation as a chain-rule-driven traversal of a computational graph. We then introduce activation checkpointing as a principled memory—computation tradeoff based on segment-wise recomputation.

We extend this framework to Transformer attention, showing how quadratic memory arises from explicit materialization of pairwise interactions. This naturally leads to FlashAttention, which can be interpreted as a structured form of online checkpointing applied to the softmax normalization over attention scores.

Finally, we unify three core systems techniques used in modern LLM training and inference:

  • Activation checkpointing (activation recomputation)

  • FSDP (parameter and optimizer state sharding)

  • KV cache (temporal state reuse in autoregressive inference)

We show that these methods are not independent optimizations, but rather different instantiations of a single principle: the tradeoff between storing, recomputing, and communicating intermediate states in a large computational graph.

Modern large language models are built on the Transformer architecture, which can be viewed as a large differentiable computational graph.

At a high level, a neural network transforms an input state through a sequence of parameterized functions:

Activation Checkpointing

Motivation

For large Transformer models, activation memory can exceed parameter memory.

However, forward computation is deterministic:

yi=fi(yi−1,Wi)\begin{equation} y_i=f_i(y_{i-1},W_i) \end{equation}yi​=fi​(yi−1​,Wi​)​​

Therefore, intermediate activations do not need to be stored permanently.

They can be recomputed from earlier states.

Segment-based checkpointing

Divide the network into kkk segments.

Assume each segment contains:

m=nk\begin{equation} m=\frac{n}{k} \end{equation}m=kn​​​

Instead of storing all activations, store only the segment boundaries:

y0,ym,y2m,...,yn\begin{equation} y_0,y_m,y_{2m},...,y_n \end{equation}y0​,ym​,y2m​,...,yn​​​

Backward computation with checkpoints

For a given segment:

  1. Load the checkpoint activation yjmy_{jm}yjm​

  2. Recompute:

yjm+1,...,y(j+1)m\begin{equation} y_{jm+1},...,y_{(j+1)m} \end{equation}yjm+1​,...,y(j+1)m​​​

  1. Perform standard backpropagation inside this segment

The missing activations are not approximated. They are exactly regenerated.

Memory contains:

  1. kkk checkpoint activations

  2. Temporary activations of one segment

Therefore:

M(k)=k+nk\begin{equation} M(k) = k+\frac{n}{k} \end{equation}M(k)=k+kn​​​

Minimizing:

dMdk=1−nk2\begin{equation} \frac{dM}{dk} = 1-\frac{n}{k^2} \end{equation}dkdM​=1−k2n​​​

k=n\begin{equation} k=\sqrt{n} \end{equation}k=n​​​

Mmin⁡=O(n)\begin{equation} M_{\min}=O(\sqrt{n}) \end{equation}Mmin​=O(n​)​​

Activation checkpointing does not eliminate the need for intermediate states.

Transformer Architecture

A Transformer layer is a composition of two main operations:

  1. Multi-head self-attention

  2. Position-wise feed-forward network (MLP)

A modern Transformer layer can therefore be viewed as:

Mixture of Experts (MoE)

Mixture of Experts architectures scale model capacity by increasing the number of parameters without increasing compute proportionally.

The key idea is to replace dense feed-forward layers with a sparse selection of expert networks.

Dense Feed-Forward Layer Baseline

In a standard Transformer block, the MLP is:

h=σ(W1x)y=W2h\begin{align} h &= \sigma(W_1 x)\\ y &= W_2 h \end{align}hy​=σ(W1​x)=W2​h​​

Typically:

W1∈Rd×dff,W2∈Rdff×d\begin{equation} W_1 \in \mathbb{R}^{d \times d_{ff}}, \quad W_2 \in \mathbb{R}^{d_{ff} \times d} \end{equation}W1​∈Rd×dff​,W2​∈Rdff​×d​​

with:

dff≈4d\begin{equation} d_{ff} \approx 4d \end{equation}dff​≈4d​​

This layer is applied to every token.

MoE Reformulation

Instead of a single MLP, we define EEE expert networks:

Experte(x)=W2eσ(W1ex)\begin{equation} \mathrm{Expert}_e(x) = W^e_2 \sigma(W^e_1 x) \end{equation}Experte​(x)=W2e​σ(W1e​x)​​

Each expert is structurally identical to a standard FFN but with independent parameters.

Routing Mechanism

A router computes a distribution over experts:

p(x)=softmax(Wrx)\begin{equation} p(x) = \mathrm{softmax}(W_r x) \end{equation}p(x)=softmax(Wr​x)​​

where:

p(x)∈RE\begin{equation} p(x) \in \mathbb{R}^{E} \end{equation}p(x)∈RE​​

Top-k routing

Instead of using all experts, only the top-kkk are activated:

S(x)=TopK(p(x))\begin{equation} \mathcal{S}(x) = \mathrm{TopK}(p(x)) \end{equation}S(x)=TopK(p(x))​​

The output becomes:

y(x)=∑e∈S(x)pe(x) Experte(x)\begin{equation} y(x) = \sum_{e \in \mathcal{S}(x)} p_e(x)\,\mathrm{Expert}_e(x) \end{equation}y(x)=e∈S(x)∑​pe​(x)Experte​(x)​​

Typically k=1k=1k=1 or k=2k=2k=2.

Why MoE increases capacity efficiently

Total parameters:

O(E⋅d⋅dff)\begin{equation} O(E \cdot d \cdot d_{ff}) \end{equation}O(E⋅d⋅dff​)​​

But compute per token:

O(k⋅d⋅dff)\begin{equation} O(k \cdot d \cdot d_{ff}) \end{equation}O(k⋅d⋅dff​)​​

Since k≪Ek \ll Ek≪E, MoE decouples:

  • parameter count

  • FLOPs per token

Differentiability Problem

The TopK operation is non-differentiable:

TopK(p(x))\begin{equation} \mathrm{TopK}(p(x)) \end{equation}TopK(p(x))​​

is discrete.

Thus gradients cannot flow through expert selection directly.

Gradient Estimation Strategies

Soft routing approximation

Replace TopK with soft mixture:

y(x)=∑e=1Epe(x) Experte(x)\begin{equation} y(x) = \sum_{e=1}^{E} p_e(x)\,\mathrm{Expert}_e(x) \end{equation}y(x)=e=1∑E​pe​(x)Experte​(x)​​

This is differentiable but expensive.

Straight-through estimator

Forward uses TopK, backward uses soft probabilities:

∂y∂p≈soft gradient\begin{equation} \frac{\partial y}{\partial p} \approx \text{soft gradient} \end{equation}∂p∂y​≈soft gradient​​

Gumbel-Softmax (conceptual alternative)

Introduce noise:

pe=exp⁡((log⁡πe+ge)/τ)∑jexp⁡((log⁡πj+gj)/τ)\begin{equation} p_e = \frac{\exp((\log \pi_e + g_e)/\tau)} {\sum_j \exp((\log \pi_j + g_j)/\tau)} \end{equation}pe​=∑j​exp((logπj​+gj​)/τ)exp((logπe​+ge​)/τ)​​​

where geg_ege​ is Gumbel noise.

As τ→0\tau \to 0τ→0, this approaches discrete sampling.

However, large-scale MoE systems typically avoid Gumbel sampling due to instability.

Load Balancing Problem

A major issue is expert collapse, where only a subset of experts is used.

Define:

Importance

Ie=∑tpe(xt)\begin{equation} I_e = \sum_{t} p_e(x_t) \end{equation}Ie​=t∑​pe​(xt​)​​

Le=∑t1[TopK(p(xt))=e]\begin{equation} L_e = \sum_{t} \mathbf{1}[\mathrm{TopK}(p(x_t)) = e] \end{equation}Le​=t∑​1[TopK(p(xt​))=e]​​

Load balancing loss

A common auxiliary loss is:

Lbalance=E∑e=1EIe⋅Le\begin{equation} \mathcal{L}_{balance} = E \sum_{e=1}^{E} I_e \cdot L_e \end{equation}Lbalance​=Ee=1∑E​Ie​⋅Le​​​

This encourages uniform usage of experts.

Expert capacity constraint

Each expert has a maximum token capacity:

C=batch sizeE⋅α\begin{equation} C = \frac{\text{batch size}}{E} \cdot \alpha \end{equation}C=Ebatch size​⋅α​​

If exceeded, tokens are dropped or rerouted.

Why MoE is difficult to train

MoE introduces:

  • routing instability

  • gradient noise from sparse activation

  • communication overhead in distributed settings

  • imbalance across experts

MoE in distributed systems

Each expert is often placed on a different device.

Thus MoE introduces:

  • all-to-all communication

  • token dispatch

  • token gather

This makes MoE both a model and systems problem.

Unified Interpretation

MoE can be viewed as:

a conditional computation mechanism that replaces dense parameter usage with a learned routing policy over a large parameter bank.

Key Insight

MoE is not simply a model architecture:

Autoregressive Inference as Sequential State Evolution

Autoregressive language models factorize sequence probability as:

p(x1,…,xT)=∏t=1Tp(xt∣x<t)\begin{equation} p(x_1, \dots, x_T) = \prod_{t=1}^{T} p(x_t \mid x_{<t}) \end{equation}p(x1​,…,xT​)=t=1∏T​p(xt​∣x<t​)​​

This induces a fundamental constraint:

Generation is inherently sequential in time.

Unlike training, tokens cannot be processed in parallel during decoding.

Why decoding is sequential

At step ttt, the model computes:

p(xt∣x1,…,xt−1)\begin{equation} p(x_t \mid x_1, \dots, x_{t-1}) \end{equation}p(xt​∣x1​,…,xt−1​)​​

Thus each step depends on all previous hidden states.

This creates a loop:

x1→x2→⋯→xT\begin{equation} x_1 \rightarrow x_2 \rightarrow \dots \rightarrow x_T \end{equation}x1​→x2​→⋯→xT​​​

Naive inference cost

Without optimization, each step recomputes all attention layers:

O(T2d)\begin{equation} O(T^2 d) \end{equation}O(T2d)​​

per generated token, leading to:

O(T3d)\begin{equation} O(T^3 d) \end{equation}O(T3d)​​

total cost for full generation.

This is computationally infeasible.

KV Cache

Key idea

Self-attention at step ttt only requires past keys and values.

Recall attention:

Attention(Q,K,V)=softmax(QKTd)V\begin{equation} \mathrm{Attention}(Q,K,V) = \mathrm{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V \end{equation}Attention(Q,K,V)=softmax(d​QKT​)V​​

At time ttt, we compute:

K1:t=all previous keysV1:t=all previous values\begin{align} K_{1:t} &= \text{all previous keys}\\ V_{1:t} &= \text{all previous values} \end{align}K1:t​V1:t​​=all previous keys=all previous values​​

We store them:

Kt=[K1,…,Kt],Vt=[V1,…,Vt]\begin{equation} \mathcal{K}_t = [K_1, \dots, K_t], \quad \mathcal{V}_t = [V_1, \dots, V_t] \end{equation}Kt​=[K1​,…,Kt​],Vt​=[V1​,…,Vt​]​​

Incremental decoding

At step t+1t+1t+1:

qt+1=xt+1WQkt+1=xt+1WKvt+1=xt+1WV\begin{align} q_{t+1} &= x_{t+1} W_Q\\ k_{t+1} &= x_{t+1} W_K\\ v_{t+1} &= x_{t+1} W_V \end{align}qt+1​kt+1​vt+1​​=xt+1​WQ​=xt+1​WK​=xt+1​WV​​​

We compute attention only over cached tensors:

ot+1=softmax(qt+1KtTd)Vt\begin{equation} o_{t+1} = \mathrm{softmax}\left( \frac{q_{t+1} \mathcal{K}_t^T}{\sqrt{d}} \right)\mathcal{V}_t \end{equation}ot+1​=softmax(d​qt+1​KtT​​)Vt​​​

Memory vs compute tradeoff

KV cache stores:

O(T⋅d)\begin{equation} O(T \cdot d) \end{equation}O(T⋅d)​​

memory per layer.

This avoids recomputation of:

O(T2d)\begin{equation} O(T^2 d) \end{equation}O(T2d)​​

terms.

Beam search

Maintain BBB candidate sequences:

{x1:t(1),…,x1:t(B)}\begin{equation} \{x_{1:t}^{(1)}, \dots, x_{1:t}^{(B)}\} \end{equation}{x1:t(1)​,…,x1:t(B)​}​​

Select highest joint probability.

Beam search is rarely used in modern LLM chat due to degeneration issues.

Speculative decoding

Motivation

Autoregressive decoding is sequential and slow.

Speculative decoding introduces a draft model to propose multiple tokens.

Two-model system

  • Small model: fast draft generator

  • Large model: verifier

Algorithm

Draft model generates:

(xt+1,…,xt+k)\begin{equation} (x_{t+1}, \dots, x_{t+k}) \end{equation}(xt+1​,…,xt+k​)​​

in parallel.

Large model evaluates them in a single forward pass.

Acceptance step

Each token is accepted with probability:

min⁡(1,plarge(x)pdraft(x))\begin{equation} \min\left(1, \frac{p_{\text{large}}(x)}{p_{\text{draft}}(x)} \right) \end{equation}min(1,pdraft​(x)plarge​(x)​)​​

Otherwise regeneration occurs.

Why it works

The key insight is:

The draft model approximates the target distribution, reducing the number of expensive large-model steps.

Even though verification is required, it is amortized over multiple tokens.

Why KV cache is still required in speculative decoding

Even when using a draft model, the large model still performs attention over past tokens.

Thus KV cache remains essential:

  • avoids recomputing past attention states

  • enables fast verification passes

Final insight

Inference in large language models is not simply evaluation of a neural network.

It is a constrained sequential decision process with:

  • persistent state (KV cache)

  • stochastic control (sampling)

  • amortized computation (speculative decoding)

All modern systems optimizations reduce to improving this sequential computation under memory and latency constraints.
